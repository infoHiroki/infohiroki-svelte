📖 よく出てくる用語解説

src (ソース)
「source（ソース）」の略 = 「元となるコード」

意味: あなたが書くプログラムのコード（元ファイル）が入っているフォルダです。

なぜsrcと略すのか: プログラマーは長い名前を嫌うので、「source」を「src」と短縮します。

中身: TypeScriptファイル（.ts）、Svelteファイル（.svelte）など、あなたが編集するファイルが入っています。

node_modules
他の人が作ったプログラム（ライブラリ）の保管庫

意味: npm install を実行したときにダウンロードされる、他の人が作ったコードの集まりです。

なぜ必要: ゼロからすべてを作るのは大変なので、便利な機能を他の人から借りてきます。例: Markdownを変換するmarked、SvelteKit本体など。

注意: このフォルダは自分で編集しません！ npm install が自動的に管理します。

削除してもいい？: はい。削除しても npm install を実行すれば再ダウンロードされます。

static
そのまま公開するファイル（画像・CSS・JSなど）

意味: 変換せずに、そのままWebサイトに公開されるファイルを入れるフォルダです。

例: 画像（.jpg, .png）、CSS（style.css）、JavaScript（main.js）など。

ルール: static/images/logo.png に置いたファイルは、Webサイトで /images/logo.png としてアクセスできます（static の部分は自動的に消えます）。

build (ビルド)
完成品のWebサイト（公開用ファイル）

意味: npm run build を実行すると作られる、最終的なWebサイトのファイルが入るフォルダです。

なぜ必要: src/ の中のTypeScriptやSvelteファイルは、ブラウザで直接実行できません。HTMLやJavaScriptに変換する必要があります。その変換後のファイルがここに入ります。

注意: このフォルダは自動生成されます！ 自分で編集してはいけません。

.svelte-kit
SvelteKitの作業用フォルダ（一時ファイル）

意味: SvelteKitが開発中に使う一時ファイルが入っています。

削除してもいい？: はい。開発サーバーを起動すると自動的に再作成されます。

注意: このフォルダは触りません！

📁 ディレクトリ構造 - 必須・オプション・自動生成の分類

SvelteKitプロジェクトのフォルダを、必須・オプション・自動生成の3つに分類して説明します。

🔴 必須ファイル・フォルダ（これがないと動かない）

📂 src/ 必須

役割: あなたが書くプログラムコードの本体

中身:

  • routes/ - ページのコード（ファイル名がURLになる）
  • lib/ - 再利用するコード（コンポーネント、関数など）
  • app.html - HTMLの土台（全ページ共通）

削除したら: プロジェクトが動きません。

📄 package.json 必須

役割: プロジェクトの設定ファイル

中身:

  • プロジェクト名・バージョン
  • 使用するライブラリのリスト（SvelteKit、marked、gray-matterなど）
  • 実行コマンド（npm run dev、npm run build など）

例:

{
  "name": "infohiroki-svelte",
  "scripts": {
    "dev": "vite dev",
    "build": "vite build"
  },
  "dependencies": {
    "marked": "^16.4.1",
    "gray-matter": "^4.0.3"
  }
}
📄 svelte.config.js 必須

役割: SvelteKitの設定ファイル

中身:

  • どのアダプターを使うか（adapter-static = 静的サイト生成）
  • ビルドの出力先（build/）
import adapter from '@sveltejs/adapter-static';

export default {
  kit: {
    adapter: adapter({
      pages: 'build',  // ビルド結果の出力先
      assets: 'build'
    })
  }
};

🔵 オプション（あると便利だが、なくても動く）

📂 static/ オプション

役割: 画像・CSS・JSなど、変換せずにそのまま公開するファイル

削除しても: 動きます。ただし、画像やCSSがなくなります。

📄 README.md オプション

役割: プロジェクトの説明書（人間が読むためのもの）

削除しても: プログラムは動きます。ただし、後で見返したときに困ります。

📄 CLAUDE.md, DESIGN.md オプション

役割: 設計思想やルールを書いたドキュメント（自分で作ったもの）

削除しても: プログラムは動きます。開発の指針として使っています。

🟡 自動生成（触ってはいけない）

📂 node_modules/ 自動生成

役割: npm install でダウンロードされたライブラリ

絶対に編集しない: 中のファイルを変更してはいけません。

削除してもいい？: はい。npm install で再生成できます。

サイズ: 数百MB〜数GBになることも。Gitには含めません（.gitignore で除外）。

📂 build/ 自動生成

役割: npm run build で作られる、公開用ファイル

絶対に編集しない: 編集しても、次のビルドで上書きされます。

削除してもいい？: はい。npm run build で再生成できます。

📂 .svelte-kit/ 自動生成

役割: SvelteKitの開発用一時ファイル

絶対に編集しない: SvelteKitが自動管理します。

削除してもいい？: はい。開発サーバー起動時に再生成されます。

📄 package-lock.json 自動生成

役割: インストールしたライブラリの正確なバージョンを記録

なぜ必要: チームで開発するとき、全員が同じバージョンのライブラリを使えるようにするため。

編集する？: 絶対に手動編集しません！ npm install が自動更新します。

🔥 Svelteとは？ - 初心者向け解説

Svelte（スヴェルト）は、Webページを作るためのツール（フレームワーク）です。HTMLを拡張して、より便利に書けるようにしたものです。

他のツール（React、Vue）との違い

React / Vue
  • ブラウザで動く仮想DOM（重い）
  • JavaScript多め、複雑
  • ファイルサイズが大きい
Svelte
  • ビルド時に最適化（速い）
  • HTMLに近い、シンプル
  • ファイルサイズが小さい

Svelteファイル（.svelte）の構造

Svelteファイルは、3つのブロックに分かれています：

<script lang="ts">
  // 1️⃣ JavaScript/TypeScriptのコード
  let count = 0;

  function increment() {
    count += 1;
  }
</script>

<!-- 2️⃣ HTML -->
<button on:click={increment}>
  クリック数: {count}
</button>

<style>
  /* 3️⃣ CSS */
  button {
    background: #ff3e00;
    color: white;
    padding: 1rem;
  }
</style>
💡 Svelteのいいところ
  • HTMLに近い: 普通のHTMLを書く感覚で使える
  • 学習曲線が緩やか: React/Vueより簡単
  • 高速: ビルド時に最適化されるので、ブラウザで高速動作
  • コード量が少ない: 同じ機能をReactより少ないコードで書ける

SvelteKitとは？

SvelteKit = Svelteを使ったWebサイト全体を作るためのツール

  • Svelte - 部品（コンポーネント）を作るツール
  • SvelteKit - Webサイト全体（ルーティング、ビルド、デプロイ）を管理するツール

例え:

  • Svelte = レゴブロック1個1個
  • SvelteKit = レゴブロックを組み合わせて家を作る設計図とツール

📘 TypeScriptとは？ - 初心者向け解説

TypeScript（タイプスクリプト）は、JavaScriptに「型」を追加した言語です。

JavaScriptとの違い

JavaScript
function add(a, b) {
  return a + b;
}

add(1, 2);      // 3
add("1", "2");  // "12" (文字列結合)

型がないので、数字と文字列が混ざってバグの原因に

TypeScript
function add(a: number, b: number): number {
  return a + b;
}

add(1, 2);      // 3
add("1", "2");  // エラー！

型があるので、間違いを書いた瞬間にエラーで教えてくれる

💡 TypeScriptのメリット
  • バグが減る: 間違いを書いた瞬間にエディタが教えてくれる
  • コードが読みやすい: 型を見れば、何を渡せばいいか一目瞭然
  • リファクタリングが安全: 型があるので、変更しても壊れにくい
  • 補完が効く: エディタが自動的にコードを補完してくれる

よく使う型の例

// 基本的な型
let name: string = "Hiroki";          // 文字列
let age: number = 30;                  // 数値
let isActive: boolean = true;          // 真偽値

// 配列
let tags: string[] = ["AI", "Svelte", "Go"];

// オブジェクト（インターフェース）
interface BlogPost {
  title: string;
  date: string;
  tags: string[];
}

let post: BlogPost = {
  title: "SvelteKit入門",
  date: "2025-10-19",
  tags: ["Svelte", "TypeScript"]
};

// null許容型
let maybePost: BlogPost | null = null;  // BlogPostかnullのどちらか

このプロジェクトでの使い方

このプロジェクトでは、src/lib/types/blog.ts で型を定義しています：

export interface BlogPost {
  slug: string;           // ファイル名
  title: string;          // 記事タイトル
  description: string;    // 説明文
  icon: string;           // アイコン絵文字
  tags: string[];         // タグ配列
  date: string;           // 日付
  content: string;        // HTML本文
}

これを使うことで、BlogPostオブジェクトの形が決まっているので、間違った形のデータを渡すとエラーになります。

🔨 ビルドとは？ - 初心者向け解説

ビルド（build） = プログラムを「完成品」に変換する作業

なぜビルドが必要？

src/ フォルダの中のファイルは、ブラウザで直接実行できません。

理由:

  • TypeScript（.ts）はブラウザが理解できない → JavaScriptに変換が必要
  • Svelte（.svelte）もブラウザが理解できない → HTML+JS+CSSに分解が必要
  • ファイルが分散している → 1つにまとめて最適化する必要がある

ビルドプロセス（何が起こっているか）

1️⃣ あなたが書いたコード
src/routes/+page.svelte
src/lib/utils/posts.ts
↓
2️⃣ Viteがビルド
TypeScript → JavaScript
Svelte → HTML+JS+CSS
↓
3️⃣ SvelteKitが静的サイト生成
全ページをHTMLファイルに出力
↓
4️⃣ 完成品
build/index.html
build/blog/index.html
（102記事分のHTMLファイル）

ビルドコマンド

開発中（プレビュー）:

npm run dev
  • 開発サーバーが起動（http://localhost:5173）
  • ファイルを編集すると、即座にブラウザに反映（HMR）
  • まだビルドしていない（build/ フォルダは作られない）

本番ビルド:

npm run build
  • 完成品を作る（build/ フォルダに出力）
  • 最適化される（ファイルサイズ削減、高速化）
  • このbuild/ フォルダをCloudflare Pagesにアップロードする

ビルド結果のプレビュー:

npm run preview
  • ビルド後のファイルをローカルで確認
  • 本番環境と同じ動作を確認できる

ビルドで何が最適化されるか

  • ファイルサイズ削減: 不要な空白・コメントを削除（minify）
  • 複数ファイルを結合: HTTPリクエスト数を減らす
  • コード分割: 必要なコードだけを読み込む（遅延ロード）
  • 型情報の削除: TypeScriptの型は実行時に不要なので削除
  • 古いブラウザ対応: 最新のJavaScript構文を古い書き方に変換
💡 ビルドのポイント
  • 開発中は npm run dev: 高速・リアルタイム更新
  • 公開前に npm run build: 最適化された完成品を作る
  • build/ フォルダは編集禁止: 次のビルドで上書きされる
  • エラーが出たら npm run check: TypeScriptの型エラーを確認

📚 まとめ

このガイドで学んだこと

  • ✅ src/ = あなたが書くコード（必須）
  • ✅ node_modules/ = ダウンロードしたライブラリ（自動生成・編集禁止）
  • ✅ static/ = 画像・CSSなど、そのまま公開するファイル（オプション）
  • ✅ build/ = ビルド後の完成品（自動生成・編集禁止）
  • ✅ Svelte = HTMLに近い、シンプルなフレームワーク
  • ✅ TypeScript = 型があるJavaScript（バグを減らす）
  • ✅ ビルド = TypeScript/Svelteを変換して完成品を作る作業
💡 次のステップ
  • 実際に src/routes/+page.svelte を編集してみる
  • npm run dev で開発サーバーを起動して、変更を確認
  • 新しいページを作ってみる（src/routes/test/+page.svelte）
  • npm run build でビルドして、build/ フォルダを確認
